Description

A server handling the low level RMCP and IPMI protocol regarding encoding and decoding of messages, as well as correct packet acknowledgement. The server basically provides API to issue RPC-style commands (a response is expected for every single request). These requests can be issued in either blocking or non-blocking manner.

Request/Response mapping is done according to the request sequence numbers provided in IPMI session wrapper. Correctness of sequence numbers has to be ensured by calling processes.

blocking_request/3

blocking_request(Pid::pid(), Request::eipmi:request(), Properties::proplists:proplist()) -> {ok, proplists:proplist(), proplists:proplist()} | {error, term()}

Sends a RPC request in a blocking manner. The call will not return until a corresponding request was received or a timeout occured.

non_blocking_request/3

non_blocking_request(Pid::pid(), Request::eipmi:request(), Properties::proplists:proplist()) -> ok

Sends a RPC-like request in a non-blocking manner. The call will return immediatelly. The response will be delivered to the calling process via an ordinary message either of the form {ipmi, {error, term()}} or {ipmi, {ok, SessionHeaderProperties, ResponseFields}}.

start_link/3

start_link(SessionHandle, Address, Port) -> any()

Starts a protocol server which in turn will occupy a free UDP socket. The provided IP address and network port will be used to send requests. This means that the target for this protocol server is static and configured on startup.
